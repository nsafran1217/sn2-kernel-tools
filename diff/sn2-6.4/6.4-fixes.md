# SN2 v6.3 → v6.4 Forward-Port Fixes

## Summary

Forward-porting `sn2-v6.3-complete.patch` (120 files, 29.8k lines) to
Linux 6.4.  The patch applies with 0 hunk failures and 7 minor line
offsets in 4 files.  One build fix is required for an API change.

All fixes are folded into the combined `sn2-v6.4-complete.patch`.

Total files: 120 (unchanged from v6.3)

## Part 1: Patch Application Fixes

The v6.3 patch applied cleanly to the 6.4 tree with no rejected hunks.
Seven hunks in four files applied with minor line offsets due to
upstream additions/removals:

### 1.1: arch/ia64/Kconfig — HAS_IOPORT added upstream

**Symptom:** Hunks #1, #2, #3 at offset +1 line.
**Cause:** Linux 6.4 added `select HAS_IOPORT` to the `config IA64`
block (line 28), shifting all subsequent lines by +1.
**Impact:** None — hunks applied successfully with offset.
**Fix:** Regenerated patch with correct context.

### 1.2: drivers/char/Kconfig — PCMCIA source removed

**Symptom:** Hunk #1 at offset -2 lines.
**Cause:** `source "drivers/char/pcmcia/Kconfig"` was removed upstream
in 6.4 (PCMCIA cleanup), shifting context.
**Impact:** None — hunk applied successfully with offset.
**Fix:** Regenerated patch with correct context.

### 1.3: drivers/tty/serial/Kconfig — Context shift

**Symptom:** Hunks #1, #2 at offset -7 lines.
**Cause:** Several serial driver entries were removed/reorganized
upstream in 6.4.
**Impact:** None — hunks applied successfully with offset.
**Fix:** Regenerated patch with correct context.

### 1.4: include/linux/pci_ids.h — New vendor IDs

**Symptom:** Hunk #1 at offset +1 line.
**Cause:** New PCI vendor IDs added upstream before our insertion point.
**Impact:** None — hunk applied successfully with offset.
**Fix:** Regenerated patch with correct context.

## Part 2: Build Fixes

### 2.1: snsc.c — class_create() API change

**Error:**
```
drivers/char/snsc.c:389: error: too many arguments to function 'class_create'
```

**Cause:** In Linux 6.4, `class_create()` was simplified to take only
the class name string, removing the `owner` parameter.  The old API was:
```c
class_create(THIS_MODULE, "name")
```
The new API is:
```c
class_create("name")
```
The `owner` field is now set automatically by the driver core.

**Fix:** Changed `class_create(THIS_MODULE, SYSCTL_BASENAME)` to
`class_create(SYSCTL_BASENAME)` in `drivers/char/snsc.c`.

**File:** `drivers/char/snsc.c`

## Part 3: Upstream Non-Breaking Changes

These are upstream changes between 6.3 and 6.4 that affect files
touched by the SN2 patch but do NOT require SN2 code changes:

- **arch/ia64/Kconfig**: `HAS_IOPORT` select added; `MAX_ORDER`
  changed from range-based to fixed defaults (0-based MAX_ORDER
  transition). Neither affects SN2 config options.

- **arch/ia64/mm/contig.c**: `__init` annotation added to
  `find_max_min_low_pfn()`. Our ZONE_DMA32 guard is in `paging_init()`
  — unaffected.

- **arch/ia64/pci/pci.c**: Documentation path changed from
  `Documentation/ia64/aliasing.rst` to
  `Documentation/arch/ia64/aliasing.rst` (comment only, no code change).

- **drivers/char/Kconfig**: PCMCIA `source` line removed. Our SGI
  config entries are in a different area — unaffected.

- **drivers/char/Makefile**: PCMCIA build line removed. Our SGI
  entries are unaffected.

- **drivers/scsi/qla1280.c**: `scsi_host_template` made `const`.
  Our `asm/machvec.h` include is in the header area — unaffected.

- **drivers/misc/sgi-xp/xpc_main.c**: Already uses new
  `register_sysctl()` API (was updated in 6.4 upstream). Not touched
  by SN2 patch.

- **include/linux/pci_ids.h**: New vendor IDs added. Our SGI PCI IDs
  are in a different location — unaffected.
