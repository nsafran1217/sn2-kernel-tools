# SN2 Re-addition: Linux 6.1 â†’ 6.2 Forward-Port Fixes

## Summary

Forward-port of `sn2-v6.1-complete.patch` to Linux 6.2. The patch
applied with only **1 rejected hunk** out of 120 files. All other
hunks applied cleanly (some with minor offsets due to upstream changes).

## Patch Application Fix

### 1. `arch/ia64/Kconfig` â€” Hunk #2 FAILED (context shift)

**Cause:** Linux 6.2 added `select FUNCTION_ALIGNMENT_32B` between
`select PCI_MSI_ARCH_FALLBACKS if PCI_MSI` and `select ZONE_DMA32`,
shifting the context for our `ZONE_DMA32` conditional.

**v6.1 patch expected:**
```
 	select PCI_MSI_ARCH_FALLBACKS if PCI_MSI
-	select ZONE_DMA32
+	select ZONE_DMA32 if !IA64_SGI_SN2
 	default y
```

**v6.2 actual context:**
```
 	select PCI_MSI_ARCH_FALLBACKS if PCI_MSI
 	select ZONE_DMA32
 	select FUNCTION_ALIGNMENT_32B
 	default y
```

**Fix:** Manually applied the same change â€” `select ZONE_DMA32 if !IA64_SGI_SN2` â€”
at the correct location (line 65 instead of 62).

## Upstream Changes Between 6.1 and 6.2 (non-breaking)

The following upstream changes in 6.2 caused offset adjustments but did
not break the patch or require code changes:

- **`arch/ia64/include/asm/io.h`**: Removed `__SLOW_DOWN_IO` / `SLOW_DOWN_IO`
  macros (4 lines removed before our SN2 block). Our SN2 I/O override block
  at the end of io.h is unaffected.

- **`drivers/Kconfig`**: Added `source "drivers/accel/Kconfig"`.

- **`drivers/char/Kconfig`**: Removed `RANDOM_TRUST_CPU` and
  `RANDOM_TRUST_BOOTLOADER` config options (moved upstream in 6.2).

- **`drivers/misc/Kconfig`** and **`drivers/misc/Makefile`**: Added
  SMPRO_ERRMON and SMPRO_MISC entries.

- **`drivers/scsi/qla2xxx/qla_init.c`**: Various upstream fixes to
  abort handling and `qla_vha_mark_busy()` rename. Our `#include
  <asm/machvec.h>` at line 20 is unaffected.

- **`drivers/tty/serial/Kconfig`**: Added `default SOC_SIFIVE || SOC_CANAAN`
  entries for SERIAL_SIFIVE.

## Build Issue Analysis

No new build issues were found for the 6.1â†’6.2 forward-port. The SN2
code compiles correctly against the 6.2 kernel APIs:

- **DMA API**: `struct dma_map_ops` unchanged between 6.1 and 6.2.
  `sn_dma_ops` uses `.map_sg`/`.unmap_sg` with 5-parameter signatures
  (including attrs), `.mmap = dma_common_mmap`, `.get_sgtable =
  dma_common_get_sgtable` â€” all still valid. `sba_iommu.c` (ia64's
  other DMA ops provider) uses the same pattern.

- **`dma_ops` extern**: Still declared as `extern const struct
  dma_map_ops *dma_ops` in `asm/dma-mapping.h` with
  `get_arch_dma_ops()` returning it. `sn_dma_init()` sets
  `dma_ops = &sn_dma_ops` â€” unchanged.

- **`DMA_ATTR_WRITE_BARRIER`**: Still defined locally in `pci_dma.c`
  as `(1UL << 0)` â€” no conflict with mainline DMA_ATTR flags.

- **Memory zones**: `CONFIG_ZONE_DMA32` is conditioned on
  `!IA64_SGI_SN2` in Kconfig. `contig.c` and `discontig.c` have
  `#ifdef CONFIG_ZONE_DMA32` guards around the DMA32 zone setup.

- **Weak symbols**: `__weak ia64_send_ipi()` in `irq_ia64.c` and
  `__weak ia64_global_tlb_purge()` in `tlb.c` â€” still valid.

- **machvec.h shim**: Provides `ia64_platform_is()` as compile-time
  check â€” still used by `acpi.c`, `mspec.c`, `qla1280.c`, and SN2
  platform code.

## Fixes Carried Forward from Previous Versions

These fixes from v5.4â†’v6.1 are present in the v6.1 patch and carry
through to 6.2 unchanged:

1. **`pci_dma.c` `DMA_ATTR_WRITE_BARRIER`**: Defined locally as
   `(1UL << 0)` with `#ifndef` guard (since v5.5).

2. **`pci_dma.c` `const sn_dma_ops`**: Uses `static const struct
   dma_map_ops` with `.mmap` and `.get_sgtable` callbacks (since v5.5).

3. **`pcibr_dma.c` `sn_dma_flush(unsigned long)`**: Uses `unsigned
   long` instead of `u64` for the addr parameter (since v5.4).

4. **`io_init.c` insert_resource error handling**: Wraps
   `insert_resource()` calls with error checking and forces
   `res->parent` on failure (since v5.5).

5. **`io_init.c` IORESOURCE_UNSET clear**: Clears `IORESOURCE_UNSET`
   flag before `insert_resource()` (since v5.4).

6. **`iomv.c` NULL check**: Checks `pda->pio_write_status_addr` for
   NULL in `__sn_mmiowb()` (since v5.4).

7. **`setup.c` simplified platform init**: Two-way `#ifdef
   CONFIG_IA64_SGI_SN2 / #else` instead of three-way with UV path
   (since v5.5).

8. **`Kconfig` SWIOTLB conditional**: `select SWIOTLB if
   !IA64_SGI_SN2` and `select ZONE_DMA32 if !IA64_SGI_SN2` (since
   v5.5/v6.1).

9. **`contig.c`/`discontig.c` ZONE_DMA32 guards**: `#ifdef
   CONFIG_ZONE_DMA32` around zone setup code (since v6.1).
