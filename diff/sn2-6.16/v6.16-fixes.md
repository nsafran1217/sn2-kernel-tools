# SN2 Patch Port: Linux 6.6 → 6.16 Fixes

## Summary

Porting the SN2 patch from Linux 6.6 to 6.16.
Starting patch: `sn2-v6.6-complete.patch` (120 files, 29,835 lines)
Result patch: `sn2-v6.16-complete.patch` (120 files, 30,210 lines)

Total changes: **7 fixes** (2 patch context drift, 2 missing files, 3 build fixes)

Note: ia64 was removed entirely from the kernel in v6.7. The 6.16
clean tree already has ia64 base support re-added. This port adds
SN2 platform support on top of that.

---

## Part 1 — Patch Application Fixes (Context Drift)

### Fix 1: `drivers/char/Makefile` — Hunk 1 rejected
The `MSPEC` entry was removed from the base kernel when ia64 was
dropped in 6.7.  The 6.16 Makefile no longer has the `MSPEC` or
`UV_MMTIMER` anchor lines in the same positions.  Manually added
`SGI_SNSC`, `MSPEC`, and `SGI_MBCS` entries in the correct positions
between `VIRTIO_CONSOLE` and `UV_MMTIMER`.

### Fix 2: `drivers/tty/serial/Makefile` — Hunks 1 & 2 rejected
The serial Makefile was reorganized into strict alphabetical order
between 6.6 and 6.16. The old anchor lines (`SERIAL_DZ`/`SERIAL_SH_SCI`
and `SERIAL_JSM`/`SERIAL_TXX9`) no longer exist in those positions.
Inserted `SERIAL_SGI_IOC4` and `SERIAL_SGI_L1_CONSOLE` alphabetically
between `SC16IS7XX_I2C` and `SH_SCI`.

## Part 3 — Build Fixes (6.6 → 6.16 API Changes)

### Fix 5: `drivers/tty/serial/sn_console.c` — uart_circ → kfifo migration
The serial core replaced `uart_state.xmit` (struct circ_buf) with
`tty_port.xmit_fifo` (DECLARE_KFIFO_PTR) between 6.6 and 6.16.
The `uart_circ_empty()`, `uart_circ_chars_pending()` helpers and
direct `xmit.head`/`xmit.tail`/`xmit.buf[]` access no longer exist.

Changes:
- Replaced `#include <linux/circ_buf.h>` with `#include <linux/kfifo.h>`
- `snp_start_tx()`: `uart_circ_empty(&port->state->xmit)` →
  `kfifo_is_empty(&port->state->port.xmit_fifo)`
- `sn_transmit_chars()`: Rewrote the dual-loop circ_buf walk into a
  `kfifo_out_linear_ptr()` / `kfifo_skip_count()` loop.  Uses
  `kfifo_is_empty()` for empty check and `kfifo_len()` for
  pending count.
- Emergency console write path: Replaced `xmit.head`/`xmit.tail`
  progress tracking with `kfifo_len()` monitoring to detect lock
  holder death.

### Fix 6: `drivers/tty/serial/ioc4_serial.c` — uart_circ → kfifo migration
Same circ_buf → kfifo migration as sn_console.c.

Changes:
- Replaced `#include <linux/circ_buf.h>` with `#include <linux/kfifo.h>`
- `transmit_chars()`: Rewrote from `state->xmit.head`/`.tail`/`.buf[]`
  circ_buf walk to `kfifo_out_linear_ptr()` / `kfifo_skip_count()`.
  Removed unused `struct tty_struct *tty` and `struct uart_state *state`
  local variables.

### Fix 7: `arch/ia64/sn/kernel/tiocx.c` — const bus_type
In 6.16, `bus_register()`, `device.bus`, and `device_driver.bus` all
require `const struct bus_type *`.  Changed `tiocx_bus_type` from
`struct bus_type` to `const struct bus_type`.

### Fix 8: `arch/ia64/sn/kernel/setup.c` — Remove module_alloc/module_memfree
The `module_alloc()` and `module_memfree()` override functions were
removed.  In 6.12, the kernel replaced these with the `execmem`
subsystem (`execmem_alloc()`/`execmem_free()`) which hardcodes
`VM_FLUSH_RESET_PERMS` in its allocation path and does not provide
an architecture hook for overriding vm_flags.

The original overrides suppressed `VM_FLUSH_RESET_PERMS` to avoid
SHUB PTC.GA TLB broadcast interference during module loading.  Since
ia64's `set_memory_nx`/`set_memory_rw`/`set_direct_map` are all
no-ops, the `vm_reset_perms()` path only performs
`flush_tlb_kernel_range()`, which may still cause some interference.
Added an explanatory comment recommending building modular drivers
as built-in if this proves problematic.

## Known Issue — SHUB PTC.GA / Module Loading

The `module_alloc()` / `module_memfree()` overrides that suppressed
`VM_FLUSH_RESET_PERMS` are no longer possible with the execmem
subsystem.  Module loading will trigger `vm_reset_perms()` →
`flush_tlb_kernel_range()` which may cause SHUB PTC.GA interference
with concurrent PIO/MMIO traffic.  If USB or other modular subsystems
exhibit crashes during loading, build them as built-in instead.
